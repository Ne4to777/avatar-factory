## ❌ Проблема #2: OpenMMLab на Windows = кошмар

### Технический контекст

MuseTalk требует OpenMMLab пакеты:
- `mmcv` (computer vision)
- `mmpose` (pose estimation)
- `mmdet` (object detection)

### Проблема установки на Windows

```python
# Попытка через pip на Windows:
pip install mmcv mmpose mmdet
```

**Результат:**
```
ERROR: Could not build wheels for mmcv
ERROR: Microsoft Visual C++ 14.0+ required
ERROR: CUDA compilation failed
ERROR: ...сотни строк ошибок...
```

### Почему не работает

1. **C++ расширения:** Требуют компиляции с MSVC
2. **CUDA integration:** Сложная настройка путей на Windows
3. **Конфликты зависимостей:** PyTorch vs OpenMMLab версии
4. **Отсутствие prebuild wheels:** Для Windows нет готовых пакетов

### Реальная оценка

**Вероятность успешной установки на Windows через pip: ~5%**

Даже опытный разработчик потратит **6-10 часов** на отладку и может не достичь успеха.

## 🎯 Почему Docker - правильное решение

### 1. OpenMMLab установка через Conda

Dockerfile использует Miniconda:

```dockerfile
# Создаём conda environment
RUN conda create -n avatar python=3.11 -y

# Устанавливаем OpenMMLab через conda-forge
RUN conda install -c conda-forge mmcv mmdet mmpose -y
```

**Conda** имеет prebuild бинарники для всех платформ → установка работает.

### 2. Изолированная среда

Контейнер включает **всё**:
- Ubuntu 22.04
- Python 3.11
- CUDA 11.8 + cuDNN
- PyTorch 2.7.0
- OpenMMLab (mmcv, mmpose, mmdet)
- MuseTalk
- Stable Diffusion XL
- Silero TTS

**Результат:** Один `docker run` → всё работает.

### 3. Воспроизводимость

Dockerfile - это **спецификация**:
- Точные версии всех пакетов
- Детерминированная установка
- Работает одинаково на всех машинах

### 4. GPU Passthrough

NVIDIA Container Toolkit пробрасывает GPU в контейнер:

```bash
docker run --gpus all ...
```

**Performance:** 90-95% от native (overhead минимальный).
